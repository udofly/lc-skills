# 分支 UI 样式与功能回归修复 (Branch UI Fix)

本 Skill 用于在性能优化、Lighthouse 优化等分支中，通过对比 master 分支的代码差异，定位并修复因优化改动而引入的 UI 样式回归和功能异常。核心原则：**优化分支的功能和视觉表现必须与 master 保持一致**。

## 适用场景

- Lighthouse / 性能优化分支，改动了 HTML 结构、CSS 类名、组件引用等
- 重构分支中样式或功能出现偏差
- 用户反馈"页面和之前不一样了"

## 执行流程

### 第一步：获取差异全貌

1. 获取当前分支名和对比目标分支（默认 master）：
   ```bash
   git branch --show-current
   git log --oneline master..HEAD | head -20
   ```

2. 获取所有变更文件列表并分类：
   ```bash
   git diff master --name-status
   ```

3. 将变更文件按类型分组：
   - **页面文件** (`.vue` in `pages/`)
   - **组件文件** (`.vue` in `components/`)
   - **样式文件** (`.less`, `.css`, `.scss`)
   - **布局/配置** (`layouts/`, `nuxt.config.ts`, `tailwind.config.js`)
   - **工具/逻辑** (`composables/`, `utils/`, `stores/`)

### 第二步：逐文件差异分析

对每个有变更的 UI 相关文件（`.vue`、`.less`、`.css`），执行以下检查：

```bash
git diff master -- <file_path>
```

**重点关注以下高风险变更模式：**

#### A. HTML 结构变更
- 删除或替换了 DOM 元素（`div` → `section` 等语义化替换是否丢失了样式绑定）
- 移除了带有样式类的包裹层（外层 `div` 被删，内部元素样式可能失效）
- `v-if` / `v-show` 条件变更导致元素不再显示
- 组件替换（如 `<van-xxx>` 换成自定义组件）是否保留了原有 props 和事件

#### B. CSS / 样式变更
- Tailwind 类名被删除或替换（特别是 `flex`、`grid`、`gap`、`p-`、`m-`、`text-`、`bg-` 等布局和视觉类）
- `.less` 文件中的选择器被删除或重命名
- 样式作用域变更（`scoped` 被移除或添加，影响子组件样式穿透）
- `!important` 被移除导致样式优先级变化
- 媒体查询或响应式断点被修改

#### C. 功能逻辑变更
- 事件绑定被移除（`@click`、`@change` 等）
- `computed` / `watch` 逻辑被修改影响渲染
- API 调用被移除或路径变更
- 路由跳转逻辑变更
- 条件渲染逻辑（`v-if`/`v-show`）条件表达式变更

#### D. 资源引用变更
- 图片路径变更或图片被删除
- 字体文件引用变更
- 第三方库引用被移除

### 第三步：问题归类与严重性评估

将发现的问题按严重性分为三级：

| 级别 | 说明 | 示例 |
|------|------|------|
| 🔴 严重 | 功能缺失或页面崩溃 | 事件绑定丢失、组件未注册、API 调用缺失 |
| 🟡 中等 | 样式明显偏差 | 布局错位、间距异常、颜色不一致、字体大小变化 |
| 🟢 轻微 | 细微视觉差异 | 圆角微调、阴影细微变化（可能是有意优化） |

### 第四步：智能修复

对每个问题执行修复，遵循以下原则：

1. **最小改动原则**：只修复回归问题，不引入额外变更
2. **保留优化成果**：如果某个改动是性能优化的一部分（如懒加载、代码分割），尽量在保留优化的前提下修复样式
3. **优先恢复 master 行为**：当优化与样式冲突时，优先保证样式正确，再寻找替代优化方案

**修复策略：**

- **样式类丢失**：补回被误删的 Tailwind 类或 LESS 样式
- **结构变更导致样式失效**：调整选择器或补充缺失的包裹元素
- **组件替换不完整**：补齐缺失的 props、events、slots
- **条件渲染异常**：恢复正确的条件表达式
- **事件绑定丢失**：恢复事件处理函数及绑定

### 第五步：生成修复报告

输出结构化报告，包含以下信息：

```markdown
## 🔧 UI 回归修复报告

### 分支信息
- 当前分支：`feature/xxx`
- 对比分支：`master`
- 变更文件数：N 个

### 发现的问题

#### 🔴 严重问题 (X 个)
1. **[文件路径]** - 问题描述
   - 原因：xxx
   - 修复：xxx

#### 🟡 中等问题 (X 个)
...

#### 🟢 轻微问题 (X 个)
...

### 修复汇总
- 已修复：X 个
- 需人工确认：X 个（列出原因）
- 有意变更（跳过）：X 个

### 建议
- 需要在真机/浏览器中重点验证的页面列表
```

## 特别注意事项

- 对于 **Vant 组件**的变更要特别小心，Vant 组件的 props 变更可能导致整个组件行为改变
- **Tailwind 与 LESS 混用**时，注意优先级冲突
- **`scoped` 样式**中如果用了 `:deep()` 穿透，结构变更可能导致穿透失效
- 如果用户指定了具体页面或文件，只分析指定范围，不要全量扫描
- 如果差异文件过多（>30个），先询问用户是否需要缩小范围或按页面逐个处理
