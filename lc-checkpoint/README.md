# lc-checkpoint

保存和恢复 Claude 会话的工作状态。把"在做什么、定了什么、还剩什么"写成检查点文件，跨会话、跨分支可恢复。

存储位置：`~/.claude/checkpoints/<项目名>/`

---

## 命令速查

| 说什么 | 效果 |
|---|---|
| `保存进度` | 以当前分支名为标题保存检查点 |
| `保存进度 登录重构` | 以指定标题保存 |
| `我上次做到哪了` / `恢复进度` | 恢复当前分支最新检查点 |
| `恢复进度 登录重构` | 按名称模糊匹配恢复 |
| `恢复进度 2` | 按列表编号恢复 |
| `列出检查点` | 列出当前分支所有检查点 |
| `列出检查点 --all` | 列出所有分支的检查点 |
| `清理检查点` | 删除 7 天前的检查点（需确认） |
| `清理检查点 --days 30` | 自定义天数清理 |

---

## 恢复优先级

没有指定名称时，按以下顺序查找：
1. 文件名匹配**当前分支名**的最新检查点
2. 以上没有 → 全局最新

---

## 典型节奏

```
# 下班前
保存进度

# 第二天新开会话
我上次做到哪了

# 多分支并行开发，切到 feature/login 后
恢复进度   ← 自动匹配 feature-login 的检查点
```

---

## 检查点格式

```markdown
---
status: in-progress
branch: feature/xxx
timestamp: 2026-07-14T16:30:00+08:00
files_modified:
  - src/auth.ts
---

## Working on: <标题>

### Summary
### Decisions Made
### Remaining Work
### Notes
```

---

## 注意事项

- 只写 `~/.claude/checkpoints/`，**绝不修改项目代码**
- 每次保存新建文件，不覆盖历史（检查点即工作日志）
- 7 天自动提示清理，手动确认后删除
